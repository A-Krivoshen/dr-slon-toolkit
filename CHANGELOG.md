# Changelog

## 0.6.1
- Выполнен hardening MVP модуля «IndexNow».
- Усилен runtime отдачи ключа `/<key>.txt` (GET/HEAD, предсказуемый ответ без лишнего вывода).
- Добавлен runtime-whitelist endpoint с безопасным fallback.
- Улучшен антидубль отправок URL для снижения конфликтов между ручной и автоматической отправкой.

## 0.6.0
- Добавлен первый безопасный MVP модуля «IndexNow».
- Добавлена ручная отправка URL из админки с nonce и проверкой прав.
- Добавлена автоотправка URL при публикации/обновлении/перемещении в корзину/удалении поддерживаемых типов записей.
- Добавлена безопасная выдача проверочного ключа IndexNow по адресу `/<key>.txt`.

## 0.5.0
- Выполнен hardening MVP модуля «REST API Control».
- Добавлен встроенный неизменяемый базовый allowlist системных маршрутов WordPress.
- Пользовательское поле системных маршрутов теперь только расширяет allowlist и не может разрушить базовые маршруты.
- Добавлены безопасные префиксные маршруты без regex для критичных сценариев редактора и медиа.

## 0.3.0
- Добавлен первый безопасный MVP модуля «REST API Control».
- Добавлены режимы управления доступом к REST API и whitelist маршрутов/namespace.
- Добавлена настройка доверенной capability и отдельный список системных маршрутов.

## 0.2.0
- Добавлена рабочая модульная архитектура с автозагрузкой Composer PSR-4.
- Добавлены центральный запуск плагина, обработчик активации и очистка при uninstall.
- Добавлена нативная страница настроек с переключателями модулей и параметрами очистки.
- Реализованы модули «Транслитерация», «Отключение комментариев» и «Очистка».
- Добавлен MVP модуля «Скрытый вход» с кастомным slug, защитой wp-login.php и аварийным bypass через `KRV_DSTK_DISABLE_HIDE_LOGIN`.
- Добавлено уведомление о совместимости с The SEO Framework.
- Улучшено объединение вложенных настроек и обработка сложных случаев slug при транслитерации.
- Снижены побочные эффекты в модулях отключения комментариев и очистки.
