# Dr.Slon Toolkit

Dr.Slon Toolkit — модульный плагин WordPress для обслуживания и базового усиления клиентских сайтов.

## Текущая версия (0.2.0)

Первая рабочая версия включает:

- минимальную точку запуска с автозагрузкой Composer (PSR-4)
- центральный класс запуска плагина
- обработчики активации и удаления
- нативную страницу настроек WordPress
- переключатели модулей через Settings API
- три рабочих безопасных модуля:
  - Скрытый вход
  - Транслитерация
  - Отключение комментариев
  - Очистка
- проверку совместимости с The SEO Framework
- единый служебный блок в админке Dr.Slon Toolkit (контакт + партнёрские материалы)

## Требования

- WordPress 6.6+
- PHP 8.1+

## Установка у клиента

Composer на сервере клиента **не нужен**.
Клиенту передаётся готовый ZIP-архив плагина, который устанавливается через стандартный установщик WordPress.

## Что уже реализовано

### Скрытый вход
- Скрывает прямой доступ к `wp-login.php` для неавторизованных пользователей.
- Открывает страницу входа по настраиваемому slug, например `/my-login/`.
- Сохраняет рабочие сценарии logout и восстановления пароля.
- Имеет аварийное отключение через константу `KRV_DSTK_DISABLE_HIDE_LOGIN` в `wp-config.php`.
- При смене slug автоматически обновляет rewrite rules.

#### Ручная проверка «Скрытого входа»
1. Включите модуль и задайте slug, например `my-login`, затем сохраните настройки.
2. Проверьте, что `wp-login.php` для гостя перенаправляется на `/my-login/`.
3. Проверьте вход, выход, `lost password` и переход по ссылке `reset password`.
4. Проверьте доступ администратора в `/wp-admin/` после входа.
5. Для аварийного доступа добавьте в `wp-config.php`:
   `define('KRV_DSTK_DISABLE_HIDE_LOGIN', true);`

### Транслитерация
- Преобразует нелатинские slug записей при необходимости.
- Преобразует нелатинские slug терминов при необходимости.
- Преобразует имена загружаемых файлов в безопасный латинский вид.
- Подключается к формированию slug при сохранении, чтобы корректно обрабатывать сложные случаи с кириллическими заголовками.

### Отключение комментариев
- Глобально закрывает комментарии и пинги.
- Убирает поддержку комментариев и трекбеков у типов записей.
- Скрывает меню комментариев и пункт комментариев в верхней панели.
- Перенаправляет экраны управления комментариями в консоль.

### Очистка
- Даёт отдельные переключатели для:
  - отключения emoji
  - отключения скрипта wp-embed
  - отключения XML-RPC
  - удаления выбранных безопасных тегов из `<head>`

## Разработка

```bash
composer install
```

После этого активируйте плагин и настройте модули в меню **Dr.Slon Toolkit** в wp-admin.

## Сборка релизного ZIP (для клиента)

Composer используется только на этапе разработки и сборки.

Вариант 1 (через Composer-скрипт):

```bash
composer build-release
```

Вариант 2 (напрямую):

```bash
bash tools/build-release.sh
```

Сценарий сборки:
- удаляет предыдущую сборку в `build/` и старый архив в `dist/`;
- ставит production-зависимости (`--no-dev`);
- формирует самодостаточный архив `dist/dr-slon-toolkit-<version>.zip`;
- упаковывает файлы внутри корневой папки `dr-slon-toolkit/`.

В архив включаются:
- `dr-slon-toolkit.php`
- `readme.txt`
- `uninstall.php`
- `src/`
- `vendor/` (включая `vendor/autoload.php`)
- `languages/` (если есть)
- `assets/` (если есть)
- `LICENSE`

Из архива исключаются:
- `.git/`, `.github/`
- `tests/`
- `AGENTS.md`
- `build/`, `dist/`
- служебные файлы редакторов, логи и прочие временные файлы
- документация разработки, не нужная для работы плагина
